One of the key ideas that I contributed to OptimumK was the idea that the software should start giving results to the user as quickly as possible. The input screen shown above is an example of that philosophy. As soon as the user puts data into the form on the left, the 3D view of the suspension on the right updates. This type of continuous feedback helps to minimize input errors and gives the user confidence that they're using the software correctly.

The above screen is another of OptimumK's input screens. In this case, the user is specifying how the vehicle will move during the analysis. In addition to interface design, I was also one of the developers of the interface and the solver algorithms.
